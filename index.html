Sprint Goals

Successfully load start up data - risk, user, firm and product by sending commands to a sequencer and generating the resulting events.

Have the sequencer generate unique IDs for users, firms and products.

Applications should listen to the event stream and create/update the necessary objects upon receipt.

Applications should track the inbound ESN and their outbound CSN.

Client Portal makes the start up data available to the website.

Client Portal authenticates web users based off of the loaded data.

Client Portal/Website should only display user data related to the user making the request. If the user is of type ADMIN, then the user can see all data.

Logger writes event data in human readable format to a log file.
